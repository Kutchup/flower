# Contributing to Flower

Thank you for your interest in contributing to Flower! This peaceful game project welcomes contributions from everyone.

## Philosophy

When contributing to Flower, please keep in mind the core philosophy:
- **Peaceful**: No violence, no combat, no stress
- **Beautiful**: Focus on aesthetics and visual appeal
- **Accessible**: Easy to understand and play
- **Joyful**: Bringing happiness to players

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion:
1. Check if the issue already exists
2. Create a new issue with a clear description
3. Include steps to reproduce (for bugs)
4. Suggest a solution if you have one

### Code Contributions

1. **Fork the repository**
2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Make your changes**
   - Follow the existing code style
   - Add comments where helpful
   - Keep changes focused and minimal

4. **Test your changes**
   - Ensure the game builds successfully
   - Test gameplay to ensure nothing breaks
   - Verify your feature works as expected

5. **Commit your changes**
   ```bash
   git commit -m "Add feature: description of your feature"
   ```

6. **Push to your fork**
   ```bash
   git push origin feature/your-feature-name
   ```

7. **Create a Pull Request**
   - Describe what your changes do
   - Reference any related issues
   - Explain why the changes are needed

## Code Style Guidelines

### C++ Style
- Use clear, descriptive variable names
- Keep functions focused on a single task
- Add comments for complex logic
- Follow existing naming conventions:
  - Classes: `PascalCase`
  - Functions/methods: `camelCase`
  - Variables: `camelCase`
  - Constants: `UPPER_SNAKE_CASE`

### File Organization
- Header files (`.h`) in `src/`
- Implementation files (`.cpp`) in `src/`
- Keep related functionality together
- Use include guards in headers

### Commit Messages
- Use clear, descriptive commit messages
- Start with a verb (Add, Fix, Update, Remove)
- Keep the first line under 50 characters
- Add details in the body if needed

## Areas for Contribution

### Easy/Beginner Tasks
- Add new flower types with different colors
- Create new pickup items
- Improve documentation
- Add code comments
- Fix typos

### Intermediate Tasks
- Add sound effects
- Implement new tool types
- Create more world features
- Improve animations
- Optimize rendering

### Advanced Tasks
- Add shader support
- Implement save/load system
- Create a level editor
- Add multiplayer support
- Implement advanced graphics features

## Testing

Before submitting a PR:
- [ ] Code builds without errors
- [ ] Code builds without warnings
- [ ] Game runs without crashes
- [ ] New features work as described
- [ ] Existing features still work

## Questions?

If you have questions about contributing:
- Open an issue with your question
- Tag it as "question"
- Be patient - we'll respond as soon as we can

## License

By contributing to Flower, you agree that your contributions will be licensed under the same license as the project.

## Code of Conduct

### Our Standards
- Be respectful and inclusive
- Welcome newcomers
- Focus on constructive feedback
- Help each other learn and grow
- Keep discussions peaceful (like the game!)

### Unacceptable Behavior
- Harassment or discrimination
- Trolling or insulting comments
- Personal attacks
- Publishing others' private information
- Any conduct that would make others uncomfortable

## Recognition

Contributors will be recognized in:
- The project README
- Release notes
- Special thanks in the game credits (future feature)
